# 配達ゲーム仕様書（MVP）

## 1. 概要

グリッドマップ上でプレイヤーを操作し、
**店舗で荷物を受け取り → 配達先へ届ける**ことでスコアを稼ぐ
時間制スコアアタックゲーム。

---

## 2. ゲームルール

### 2.1 基本ループ

1. 店舗で荷物を取得
2. 配達先へ移動
3. 配達完了でスコア加算
4. 新しい配達先が生成される
5. 制限時間まで繰り返し

---

### 2.2 勝敗条件

* 制限時間終了時のスコアで評価

---

## 3. マップ仕様

| 項目  | 内容      |
| --- | ------- |
| 形式  | グリッド    |
| サイズ | 10 × 10 |
| 座標  | (x, y)  |

---

## 4. オブジェクト

### 4.1 プレイヤー

* 初期位置：任意（中央推奨）
* 移動：上下左右1マス

---

### 4.2 店舗

* 数：5箇所（固定）
* 配置：固定座標
* **各配達ごとに「指定店舗」が1つ選ばれる**

* 機能：
  - 指定店舗に到達すると荷物を取得できる
  - 指定されていない店舗では荷物は取得できない

例：

```
(1,1)
(1,8)
(5,5)
(8,1)
(8,8)
```

* 機能：荷物を取得

---

### 4.3 配達先

* 常に1つ存在
* ランダム生成
* 店舗と重複しない

---

### 4.4 指定店舗

* 毎回ランダムに1つ選ばれる
* 配達開始時に決定される
* 配達完了後に再抽選される
* UI上でプレイヤーに分かるように表示する（色変更など）

---

## 5. 状態管理（State）

```text
playerPosition
storePositions
targetPosition
hasPackage
score
timeLeft
```

---

## 6. ゲームロジック

### 6.1 プレイヤー移動

```text
movePlayer(direction)
```

---

### 6.2 荷物取得

```text
if playerPosition ∈ storePositions:
    hasPackage = true
```

---

### 6.3 配達処理

```text
if playerPosition == targetPosition AND hasPackage:
    score += 1
    hasPackage = false
    spawnTarget()
```

---

### 6.4 配達先生成

```text
spawnTarget():
    ・ランダムなマスに生成
    ・店舗と重複しない
    ・プレイヤー位置と重複しない（推奨）
```

---

### 6.5 タイマー

* 初期値：60秒
* 毎秒カウントダウン
* 0秒でゲーム終了

---

## 8. 制約条件

* 配達は荷物所持時のみ可能
* 店舗と配達先は重複不可
* 配達先は常に1つ
